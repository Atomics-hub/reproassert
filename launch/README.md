# Approval-gated launch kit

This directory contains drafts and operating checklists. Nothing here authorizes a model call,
maintainer contact, third-party post, pull request, issue comment, paid service, or Marketplace
agreement.

Use the documents in this order:

1. [`benchmark-release-checklist.md`](benchmark-release-checklist.md) closes the technical and
   evidence gates.
2. [`spend-authorization-template.md`](spend-authorization-template.md) turns any proposed model
   run into one explicit, capped decision.
3. [`maintainer-validation-packet.md`](maintainer-validation-packet.md) packages a completed
   candidate for independent review.
4. [`outreach-drafts.md`](outreach-drafts.md) is copy only. Sending it requires separate exact
   approval.

The current benchmark score and maintainer-validation count remain unchanged until canonical
evidence is committed through the benchmark finalizer and an external maintainer actually responds.
